# Project 4 requirement traceability

This document maps the supplied **Artificial Intelligence Project 4 - Image or
Text Recognition (Basic)** brief to repository evidence.

DecodeVision implements the OCR path described by the blueprint.

| Blueprint requirement | Implementation evidence |
|---|---|
| Use a pre-trained model or simple library | `src/vision/ocr.py` integrates Google's native Tesseract engine through a narrow `pytesseract` adapter. |
| Perform recognition on sample input | `data/samples/decodevision_invoice.png` is a deterministic, versioned document generated by `scripts/generate_sample.py`. The UI also accepts PNG/JPG uploads. |
| Display output clearly | `app.py` presents confidence-filtered text, word-level evidence, accepted/rejected boxes and TXT/JSON downloads. |
| Input → process → output architecture | Image bytes are decoded, preprocessed, recognized and translated into domain models before presentation. |
| Grayscale conversion | `preprocess_image` collapses BGR input to a two-dimensional intensity matrix. |
| Gaussian blur | A configurable odd kernel removes micro-noise before segmentation. |
| Adaptive thresholding | Adaptive Gaussian and Otsu modes produce explicit binary foreground/background matrices. |
| Deskewing | Foreground geometry estimates a safe rotation angle and aligns the final OCR matrix. |
| Page segmentation modes | PSM 3, 6, 7 and 11 are validated by the adapter and explained in the interface. |
| Formatted text strings | Word metadata is rebuilt by block, paragraph and line identifiers. |
| Confidence scores | Every word keeps the native Tesseract confidence returned by `image_to_data`. |
| 80% confidence filter | The default gate is 80; only words meeting the active threshold become validated output. |
| Library integration check | A successful `PipelineResult` proves the native adapter returned structured data; missing Tesseract produces an explicit setup boundary instead of fake output. |
| Preprocessing integrity check | Every intermediate matrix is inspectable in the UI and verified by automated tests. |
| Accuracy benchmarking check | The validation workspace reports mean accepted confidence against the configured gate. |
| Visual confirmation check | Accepted and rejected word boxes are drawn with coordinates from the OCR response. |
| Quality verification | `tests/test_decodevision.py` covers input validation, both threshold modes, deskew behavior, parsing, filtering, orchestration, overlays and all four milestone checks. |

## Honest verification boundary

The native Tesseract executable is an operating-system dependency and is not
vendored into this repository. Automated tests validate the full application
contract with deterministic Tesseract-shaped output; a live OCR run additionally
requires Tesseract and its English language data to be installed.
